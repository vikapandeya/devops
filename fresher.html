💡 Linux is the backbone of DevOps. Every server, container, and CI runner runs Linux. Master this first.

File System Hierarchy

/ ├── bin/ # Essential user binaries (ls, cat, cp) ├── sbin/ # System binaries (ifconfig, fdisk) ├── etc/ # Configuration files (/etc/hosts, /etc/fstab) ├── home/ # User home directories ├── var/ # Variable data (logs, databases) ├── tmp/ # Temporary files (cleared on reboot) ├── usr/ # User programs (/usr/bin, /usr/lib) ├── proc/ # Virtual FS for process info (read-only) └── dev/ # Device files (disks, terminals)

File Permissions Deep Dive

# Permission format: -rwxrwxrwx  (type)(owner)(group)(others)
# r=read(4)  w=write(2)  x=execute(1)

ls -la file.txt
# -rw-r--r-- 1 user group 1234 Jan 1 10:00 file.txt

chmod 755 script.sh      # rwxr-xr-x — owner full, others read+exec
chmod 644 config.txt     # rw-r--r-- — owner read+write, others read only
chmod +x deploy.sh       # add execute for all
chmod u+rw,go-w file     # symbolic mode

chown user:group file    # change owner
chown -R user:group dir/ # recursive

Text Processing — grep / awk / sed / cut

# ── grep ── search inside files
grep "error" /var/log/app.log             # basic search
grep -i "error" app.log                   # case-insensitive
grep -r "TODO" ./src/                     # recursive search
grep -n "fail" app.log                    # show line numbers
grep -v "info" app.log                    # invert (exclude matches)
grep -E "error|warn|fatal" app.log        # extended regex (multiple patterns)
grep -c "error" app.log                   # count matches
grep -A 3 -B 2 "exception" app.log        # 3 lines after, 2 lines before
grep --color "error" app.log              # highlight matches

# ── awk ── column/field processing
awk '{print $1}' file.txt                 # print first column
awk '{print $1, $3}' file.txt             # columns 1 and 3
awk -F: '{print $1}' /etc/passwd          # custom delimiter (:)
awk '{sum += $1} END {print sum}' nums    # sum first column
awk '/error/ {print NR, $0}' app.log      # line number + full line for errors
ps aux | awk '{print $1, $2, $3}' | head  # process user, pid, cpu

# ── sed ── stream editor (find & replace)
sed 's/foo/bar/' file.txt                 # replace first occurrence per line
sed 's/foo/bar/g' file.txt                # replace ALL occurrences
sed -i 's/foo/bar/g' file.txt             # edit file IN PLACE
sed -n '10,20p' file.txt                  # print lines 10-20
sed '/^#/d' config.txt                    # delete comment lines
sed 's/^/  /' file.txt                    # add 2-space indent to each line

# ── cut / sort / uniq / wc ──
cut -d: -f1 /etc/passwd                   # cut field 1, delimiter :
cut -c1-10 file.txt                       # cut characters 1-10
sort file.txt                             # alphabetical sort
sort -n numbers.txt                       # numeric sort
sort -rn numbers.txt                      # reverse numeric sort
sort -k2 file.txt                         # sort by column 2
uniq file.txt                             # remove adjacent duplicates
sort file.txt | uniq -c | sort -rn        # count + rank occurrences
wc -l file.txt                            # count lines
wc -w file.txt                            # count words

[Install]

# Reload systemd and enable

sudo systemctl enable --now myapp

Compression & Archives

# ── tar (most common in Linux) ──
tar -czf archive.tar.gz /path/to/dir/    # compress (c=create, z=gzip, f=file)
tar -cjf archive.tar.bz2 /path/to/dir/  # bzip2 (smaller, slower)
tar -xzf archive.tar.gz                  # extract
tar -xzf archive.tar.gz -C /dest/        # extract to specific dir
tar -tzf archive.tar.gz                  # list contents without extracting
tar -xzf archive.tar.gz specific-file   # extract single file

# ── gzip / gunzip ──
gzip file.txt          # compress: creates file.txt.gz, removes original
gunzip file.txt.gz     # decompress
gzip -k file.txt       # keep original (--keep)

# ── zip / unzip (cross-platform) ──
zip -r archive.zip dir/    # recursive zip
unzip archive.zip          # extract
unzip -l archive.zip       # list contents
unzip archive.zip -d /dest # extract to destination

# ── Backup pattern ──
DATE=$(date +%Y%m%d)
tar -czf "/backup/app-${DATE}.tar.gz" /opt/myapp
find /backup -name "app-*.tar.gz" -mtime +30 -delete  # purge >30 days

Shell Scripting — Intermediate

#!/bin/bash
set -euo pipefail   # exit on error, undefined var, pipe failure

# ── Arrays ──
servers=("web01" "web02" "db01")
echo "${servers[0]}"               # first element
echo "${servers[@]}"               # all elements
echo "${#servers[@]}"              # count (3)
for s in "${servers[@]}"; do
  echo "Checking $s..."
  ssh "$s" "uptime"
done

# ── String operations ──
str="hello-world.tar.gz"
echo "${str^^}"                    # HELLO-WORLD.TAR.GZ (uppercase)
echo "${str,,}"                    # hello-world.tar.gz (lowercase)
echo "${#str}"                     # length: 19
echo "${str%.*}"                   # remove last extension: hello-world.tar
echo "${str%%.*}"                  # remove all after first dot: hello-world
echo "${str#*-}"                   # remove up to first dash: world.tar.gz
echo "${str/world/earth}"          # replace: hello-earth.tar.gz

# ── Arithmetic ──
x=10; y=3
echo $((x + y))     # 13
echo $((x % y))     # 1 (modulo)
echo $((x ** y))    # 1000 (power)

# ── Functions with return values ──
check_port() {
  local host=$1 port=$2
  nc -z "$host" "$port" 2>/dev/null
  return $?                        # 0=open, 1=closed
}

if check_port "localhost" 3306; then
  echo "MySQL is up"
else
  echo "MySQL is down" >&2
  exit 1
fi

# ── Read user input ──
read -p "Enter environment [dev/prod]: " ENV
read -s -p "Enter password: " PASS   # -s = silent (no echo)
echo

# ── Parse arguments ──
while [[ "$#" -gt 0 ]]; do
  case $1 in
    -e|--env)   ENV="$2"; shift ;;
    -v|--verbose) VERBOSE=1 ;;
    *) echo "Unknown: $1"; exit 1 ;;
  esac
  shift
done

Firewall — UFW & iptables

# ── UFW (Ubuntu Uncomplicated Firewall) ──
sudo ufw status verbose           # show rules
sudo ufw enable                   # enable firewall
sudo ufw disable                  # disable

sudo ufw allow 22/tcp             # allow SSH
sudo ufw allow 80/tcp             # allow HTTP
sudo ufw allow 443/tcp            # allow HTTPS
sudo ufw allow from 10.0.0.0/24   # allow entire subnet
sudo ufw deny 3306/tcp            # block MySQL from outside
sudo ufw delete allow 80/tcp      # remove rule

# ── iptables (lower level) ──
iptables -L -n -v                  # list all rules with counts
iptables -A INPUT -p tcp --dport 22 -j ACCEPT     # allow SSH in
iptables -A INPUT -p tcp --dport 80 -j ACCEPT     # allow HTTP in
iptables -A INPUT -j DROP          # drop everything else

# ── Save/restore iptables ──
iptables-save > /etc/iptables/rules.v4
iptables-restore < /etc/iptables/rules.v4
bash

IP Addressing & CIDR

# CIDR notation: IP/prefix-length
192.168.1.0/24   → 256 addresses  (.0 network, .255 broadcast, .1-.254 usable)
10.0.0.0/16      → 65536 addresses
10.0.0.0/8       → 16.7M addresses (class A private)

# Common private ranges (RFC 1918):
10.0.0.0/8       # large enterprise networks
172.16.0.0/12    # Docker default bridge, medium networks
192.168.0.0/16   # home networks

# Subnetting quick math:
# /24 → 2^(32-24) = 256 addresses
# /25 → 128 addresses (split /24 in half)
# /28 → 16 addresses (common for small subnets)
# /32 → 1 address (single host)

# ipcalc — subnet calculator
ipcalc 192.168.1.0/24    # shows netmask, broadcast, range

Resolving Merge Conflicts

# 1. Attempt merge
git merge feature/login
# CONFLICT — Auto-merging app.py failed

# 2. Open conflicted file — it looks like this:
# <<<<<<< HEAD  (your current branch)
# login_attempts = 3
# =======       (divider)
# login_attempts = 5
# >>>>>>> feature/login  (incoming branch)

# 3. Edit file: keep the version you want, delete markers
login_attempts = 5   # keep this

# 4. Stage and commit
git add app.py
git commit -m "fix: resolve login attempts conflict"

D. SSH & Key Management

Security ⏱ ~25 min
💡 SSH is how you securely access every server, cloud VM, and remote system. Master this before touching Docker or CI/CD.

SSH Key Setup (Do This Once)

# 1. Generate a key pair (Ed25519 is modern and secure)
ssh-keygen -t ed25519 -C "your@email.com"
# ► Saves private key: ~/.ssh/id_ed25519   (NEVER share this)
# ► Saves public key:  ~/.ssh/id_ed25519.pub (share this freely)

# 2. View your public key (paste this into servers/GitHub)
cat ~/.ssh/id_ed25519.pub

# 3. Copy public key to a remote server
ssh-copy-id -i ~/.ssh/id_ed25519.pub user@192.168.1.100
# Or manually: append to ~/.ssh/authorized_keys on the server

# 4. Connect to server
ssh user@192.168.1.100                    # default port 22
ssh -p 2222 user@192.168.1.100            # custom port
ssh -i ~/.ssh/id_ed25519 user@server.com  # specify key

# 5. SSH config file — shortcuts
# ~/.ssh/config
Host myserver
    HostName 192.168.1.100
    User ubuntu
    IdentityFile ~/.ssh/id_ed25519
    Port 22

ssh myserver   # now this is all you need to type!
bash

SSH Tunneling & Port Forwarding

# Local port forward — access remote service locally
# Access remote DB (port 5432) as if it's on localhost:5433
ssh -L 5433:localhost:5432 user@server.com

# Remote port forward — expose local service to remote server
ssh -R 8080:localhost:3000 user@server.com

# SOCKS proxy — route all browser traffic through server
ssh -D 1080 user@server.com

SSH Server Hardening

# /etc/ssh/sshd_config — secure defaults
PermitRootLogin no               # never allow root login via SSH
PasswordAuthentication no        # key-only, no passwords
PubkeyAuthentication yes
Port 2222                        # non-default port (minor security)
AllowUsers deploy ubuntu         # whitelist specific users
MaxAuthTries 3                   # lockout after 3 failures

# Restart SSH after changes
sudo systemctl restart sshd

# Key permissions — SSH is strict about this
chmod 700 ~/.ssh
chmod 600 ~/.ssh/id_ed25519      # private key: owner read-only
chmod 644 ~/.ssh/id_ed25519.pub  # public key: world-readable OK
chmod 600 ~/.ssh/authorized_keys

Automation ⏱ ~35 min

CI/CD Pipeline Flow

Developer Push │ ▼ [ CI ] Code Checkout → Install → Lint → Test → Build → Docker Build → Push to Registry │ ▼ [ CD ] Pull Image → Deploy to Staging → Smoke Test → Deploy to Production │ ▼ [ Monitor ] Alerts if errors spike → Rollback if needed

GitHub Actions — Your First Pipeline

# .github/workflows/ci.yml
name: CI Pipeline

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest    # VM that runs the job

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Run linter
        run: npm run lint

      - name: Run tests
        run: npm test

      - name: Build app
        run: npm run build

  docker:
    needs: test               # runs only if test job passes
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Build Docker image
        run: docker build -t myapp:${{ github.sha }} .

      - name: Push to Docker Hub
        run: |
          echo "${{ secrets.DOCKER_PASSWORD }}" | docker login -u "${{ secrets.DOCKER_USERNAME }}" --password-stdin
          docker push myapp:${{ github.sha }}

# ── Matrix builds ── test on multiple versions
jobs:
  test:
    strategy:
      matrix:
        node: [18, 20, 22]
        os: [ubuntu-latest, macos-latest]
    runs-on: ${{ matrix.os }}
    steps:

          node-version: ${{ matrix.node }}
      - run: npm test

      - uses: actions/cache@v4

          path: ~/.npm
          key: ${{ runner.os }}-node-${{ hashFiles('**/package-lock.json') }}
          restore-keys: |
            ${{ runner.os }}-node-

      - uses: actions/upload-artifact@v4
        with:
          name: build-output
          path: dist/

    needs: test

      - uses: actions/download-artifact@v4
        with:
          name: build-output
          path: dist/

on:
  workflow_dispatch:

        type: boolean

Interview Prep — Fresher Level

Interview ⏱ ~25 min
What is DevOps and why is it important? ▼
DevOps is a culture and set of practices that combine software development (Dev) and IT operations (Ops) to shorten the development lifecycle and deliver high-quality software continuously. It's important because it breaks silos between dev and ops teams, enables faster deployments (from months to hours), improves reliability through automation, and allows rapid response to business needs. Companies like Netflix deploy hundreds of times per day using DevOps practices.
Explain the difference between rm and rm -rf in Linux ▼
rm file.txt removes a single file. rm -r dir/ removes a directory recursively. rm -f file force-removes without prompting. rm -rf dir/ combines both: recursively removes everything in the directory without any confirmation. It is one of the most dangerous commands in Linux — rm -rf / would delete your entire filesystem. Always double-check the path before running it.
What is the difference between a Docker image and a Docker container? ▼
A Docker image is a read-only blueprint/template that contains the application code, runtime, libraries, and dependencies. It's built from a Dockerfile and stored in a registry. A Docker container is a running instance of an image — it's the image brought to life with a writable layer on top. You can run many containers from the same image (like running multiple copies of your app). Think of the image as a cookie cutter and containers as the cookies.
What is CI/CD? What is the difference between CI and CD? ▼
CI (Continuous Integration): Automatically integrating code changes from multiple developers into a shared repository, running tests on every push to detect issues early. CD (Continuous Delivery): Automatically deploying code to a staging environment after CI passes — requires manual approval for production. CD (Continuous Deployment): Fully automated — every change that passes tests goes straight to production without human approval. The key distinction is the manual gate between delivery and deployment.
What is DNS and how does it work? ▼
DNS (Domain Name System) translates human-readable domain names (google.com) into IP addresses (142.250.185.78). When you type a URL: 1) Browser checks local cache → 2) OS checks /etc/hosts → 3) Query sent to configured DNS resolver (usually your ISP's) → 4) Resolver queries root servers → TLD servers (.com) → authoritative nameservers → 5) Returns IP address → 6) Browser connects to that IP. TTL (Time-To-Live) controls how long entries are cached.
Explain Git merge vs Git rebase ▼
Merge: Creates a new "merge commit" joining two branches, preserving the complete history of both branches. The history shows exactly when branches diverged and merged. Use merge for public/shared branches. Rebase: Moves (replays) your commits on top of another branch, creating a linear history as if you had started your work from the latest commit of the target branch. Cleaner history but rewrites commits. Use rebase for local feature branches before merging. Golden rule: never rebase shared/public branches.
What are environment variables and why are they used? ▼
Environment variables are key-value pairs stored outside your application code that configure the app's behavior at runtime. They're used to: 1) Keep secrets (passwords, API keys) out of code and version control. 2) Allow the same Docker image to run in dev/staging/prod with different configs. 3) Make the app "12-factor app" compliant. Examples: DATABASE_URL, NODE_ENV, API_KEY. In Docker: docker run -e NODE_ENV=production myapp. In GitHub Actions: stored as encrypted secrets.
What is the use of .gitignore? ▼
The .gitignore file tells Git which files and directories to not track or commit. This is critical for: 1) Keeping secrets safe — never commit .env files with passwords. 2) Reducing repo size — don't commit node_modules/ (thousands of files). 3) Avoiding OS clutter — ignore .DS_Store (Mac) or Thumbs.db (Windows). Common entries: node_modules/, *.log, .env, dist/, *.pyc. If you accidentally commit a file, git rm --cached filename stops tracking it.
